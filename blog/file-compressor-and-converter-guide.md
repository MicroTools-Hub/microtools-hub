## Understanding File Compression

### What Is File Compression?

File compression reduces file size using algorithms that eliminate redundancy and unnecessary data. There are two types:

**Lossless Compression:**
- No data is permanently removed
- Files can be restored to original quality
- Best for: Documents, spreadsheets, archives
- Typical reduction: 30–50%

**Lossy Compression:**
- Some data is discarded (imperceptible to humans)
- File size reduction is more aggressive
- Best for: Images, video, audio
- Typical reduction: 50–90%

## Compression Tips by File Type

### Images

**PNG → JPG:**
- JPG is smaller (20–50% reduction) but loses transparency
- Best for photos and non-transparent images

**JPG → WebP:**
- 25–35% smaller than JPG at same quality
- Not supported on older browsers/devices

**Optimal PNG settings:**
- Use PNG only if transparency is essential
- For flat-color graphics, PNG works great
- For photos, JPG or WebP is better

### Documents (PDF, Word)

- Remove embedded images or convert to lower resolution
- Use compression tools to eliminate redundant data
- Expect 20–40% reduction without quality loss

### Archives (ZIP, RAR)

- Select "high compression" algorithms
- Expect 40–70% reduction depending on file contents
- Lossless compression—no quality loss

## Pro Tips for File Optimization

1. **Choose the right format first**: Don't convert multiple times (each conversion can degrade quality)
2. **Balance compression with quality**: Use medium compression for most use cases
3. **Test before sharing**: Download and preview compressed files before sending to others
4. **Archive old files**: Compress and backup large files to external storage
5. **Monitor cloud storage**: Regularly compress files to free up cloud space

## Common Compression Mistakes to Avoid

1. **Over-compressing images**: Heavy compression creates visible artifacts (pixelation, color banding)
2. **Converting lossy formats multiple times**: Never convert JPG → PNG → JPG (compounds quality loss)
3. **Wrong format for the task**: Using JPG for graphics with text (blurry results) instead of PNG
4. **Ignoring file requirements**: Some platforms require specific formats—check before converting
5. **Not backing up originals**: Always keep original files before aggressive compression
