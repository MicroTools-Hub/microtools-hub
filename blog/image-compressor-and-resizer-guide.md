## Understanding Image Compression and Resizing

### Image Compression Basics

Compression reduces file size using one of two methods:

**Lossless Compression:**
- No visible quality loss
- Uses PNG or WebP formats
- Best for: Graphics, logos, text
- Size reduction: 10–30%

**Lossy Compression:**
- Slight quality loss (imperceptible at normal viewing)
- Uses JPG or WebP formats
- Best for: Photographs, complex images
- Size reduction: 50–80%

### Image Resizing

Resizing changes image dimensions (width × height):
- **Scaling down**: Reduces file size and dimensions simultaneously
- **Scaling up**: Increases size; quality degrades (use carefully)
- **Aspect ratio**: Maintaining proportions prevents distortion

## Frequently Asked Questions

**Q: Will compression make my images look bad?**

A: No. Modern compression uses algorithms that preserve imperceptible detail. You won't notice quality loss unless you're comparing side-by-side at 100% zoom.

**Q: What's the difference between resizing and cropping?**

A: Resizing changes dimensions while preserving the full image. Cropping removes parts of the image. This tool focuses on resizing and compression.

**Q: Can I resize images without losing quality?**

A: Yes, for scaling *down* (making smaller). Scaling *up* (making larger) causes quality loss because pixels are enlarged.

**Q: What's the best format for web images?**

A: WebP (if your audience uses modern browsers), otherwise JPG. PNG is best for images requiring transparency.

**Q: How much can I compress without visible loss?**

A: JPG: 40–60% reduction is safe; PNG/WebP: 30–40% reduction. Results vary by image content.

## Pro Tips for Image Optimization

1. **Use JPG for photographs**: Lower overhead than PNG for photo content
2. **Use PNG for graphics and logos**: Supports transparency; prevents quality loss
3. **Use WebP for modern websites**: 25–35% smaller than JPG, excellent quality
4. **Resize before compressing**: Smaller dimensions = smaller files
5. **Batch optimize**: Use batch tools for many images at once
6. **Test across devices**: Verify compressed images look good on phones and desktops
7. **Archive originals**: Keep high-quality originals for future use

## Common Image Optimization Mistakes

1. **Over-compressing photographs**: Aggressive JPG compression creates visible artifacts
2. **Using PNG for photographs**: Unnecessarily large file sizes
3. **Huge dimensions for web**: 4000×3000 pixel images are overkill for web (wastes bandwidth)
4. **Not using WebP**: Missing 30%+ size savings for modern browsers
5. **Compressing already-compressed images**: Repeated JPG compression compounds quality loss
6. **Ignoring aspect ratio**: Distorted images look unprofessional
7. **No mobile optimization**: Desktop-optimized images are too large for phones
